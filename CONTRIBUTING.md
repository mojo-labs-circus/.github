# Contributing to mojo-labs

Thanks for your interest. mojo-labs is in early design and development — contributions are welcome but the project is moving fast and priorities shift frequently. Reading the [framework wiki](https://github.com/mojo-labs-circus/mojo/wiki) first will save you time.

## Before you start

- Check the open issues in the relevant repo before starting work
- For anything non-trivial, open an issue to discuss the approach first — PRs that land without prior discussion often get rejected not because they're wrong but because the direction shifted
- The framework sessions in the [mojo wiki](https://github.com/mojo-labs-circus/mojo/wiki) define the current design intent; don't work against it

## How to contribute

1. Fork the repo and create a branch from `main`
2. Make your changes — keep the diff focused and minimal
3. If your change affects architecture or introduces a new design decision, note it clearly in the PR description
4. Open a pull request with a clear title and description of what and why

## Code style

- Follow the conventions in the existing codebase
- Prefer simple and explicit over clever and abstract
- No new dependencies without a strong reason

## Reporting bugs

Open an issue in the relevant repo. Include:

- What you expected to happen
- What actually happened
- Steps to reproduce
- Environment (OS, relevant versions)

## Security vulnerabilities

Do not open a public issue. See [SECURITY.md](SECURITY.md).
